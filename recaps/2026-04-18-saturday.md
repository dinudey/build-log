## This week's nudge

No experiments shipped this week. Here are three specific things worth building in the next 1–2 hours:

**1. ADHD reading companion (Claude API + plain HTML)**
The impulse that started and got deleted deserves a real try. Build a single-page tool: paste in any dense text, get back three versions — a bullet summary, an ELI15 explanation, and a "what's the actual point?" one-liner. No backend needed, just a fetch call to the Claude API and a bit of layout. The real experiment: does the format actually reduce re-reading time, or does it just feel like it does? Ship it and use it on one real article to find out.

**2. Claude Code session log hook**
Build a session-end hook that appends a one-line structured entry to a local markdown file whenever a Claude Code session closes. Format: `[timestamp] | [repo] | [one-sentence description of what changed]`. Run it for a week. The experiment is whether passive capture reveals patterns in your own usage that you wouldn't otherwise notice — not the hook itself, but what it shows you.

**3. Constraint-first component design**
Pick one UI component — a notification banner, a settings toggle, a status indicator — and design it three ways: (a) default/obvious, (b) no text labels allowed, (c) one color only. Feed all three to Claude with a prompt asking it to evaluate which communicates intent most clearly and why. The experiment is less about the component and more about using Claude as a design critic — learning where that's genuinely useful and where it's confidently wrong.
